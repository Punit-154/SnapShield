- **AI language**: Used phrases like "dive into a world of diverse animal themes" and "perfect escape from the daily hustle"

> [!WARNING]
> **Key Issue**: When generating KDP listings for animal coloring books as a set, the model tends to hallucinate specific product details (page counts, paper quality) and use promotional AI language.

> [!IMPORTANT]
> **Pattern Detected**: The model consistently hallucates specific page/design counts for KDP book descriptions when not explicitly provided. This is the #1 issue to fix.

---

## Recommendations

### Immediate (Fix in prompt templates)
1. **Always specify exact page count in the prompt** — prevents hallucination of design counts
2. **Add "Do not invent product specifications"** to the system prompt  
3. **Add banned phrases list** to the system prompt to prevent AI-sounding language

### Medium-term (Model/pipeline improvements)
4. **Post-generation validation**: Check output for hallucinated numbers that don't match input
5. **Consider fine-tuning a LoRA** on high-quality listing examples after collecting 50+ good samples
6. **Run ASSERT periodically** to detect quality regression as prompts are updated

### Prompt Template Fix
Add this to the KDP system prompt:
```
CRITICAL RULES:
- NEVER invent page counts, design counts, or paper specifications unless explicitly provided
- NEVER use these phrases: "dive into", "perfect escape", "embark on", "world of"
- Every product detail you mention MUST be taken directly from the input provided
```
